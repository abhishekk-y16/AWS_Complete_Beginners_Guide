# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this documentation repository, please report it by:

1. **DO NOT** open a public issue
2. Email the maintainers at security@awsbeginnersguide.dev
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Scope

This repository contains documentation only. Security concerns typically relate to:

- Incorrect security advice in documentation
- Code examples with security vulnerabilities
- Exposed credentials or sensitive information
- Malicious links or content

## Response Time

- Initial response: Within 48 hours
- Status update: Within 7 days
- Resolution: Depends on severity

## Supported Versions

We maintain security for:

- Latest main branch
- Last 2 major releases (if applicable)

## Security Best Practices

When contributing code examples:

- Never include real AWS credentials
- Use placeholder values clearly marked as such
- Follow AWS security best practices
- Include security warnings where appropriate

Thank you for helping keep this project secure!
